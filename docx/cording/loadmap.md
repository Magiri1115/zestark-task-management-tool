## フロントエンドの実装順

1. ルーティングと画面スケルトン整備
   - `/login`, `/tasks`, `/` リダイレクト、404／error 画面を定義する
   - 画面遷移設計書どおりの URL と認証要件を満たす

2. タスク一覧画面 UI（モックベース）
   - 現在の `/tasks` のモック画面をベースにレイアウトを完成させる
   - プロジェクト概要カード、完了率グラフ、タスク一覧テーブルを設計書に合わせる

3. 認証・ロール連携
   - `next-auth` によるセッション情報をフロントで参照できるようにする
   - admin / viewer に応じてボタン表示・編集可否を切り替える

4. DB 連動と Task 一覧の実データ化
   - `getTasks` などのサーバーアクションと接続し、モックを廃止する
   - 集計値（総タスク数、完了率、工数など）をフロント側で算出・表示する

5. タスク編集モーダル（TASK-02）
   - `/tasks?edit={id}` でモーダルを開閉する実装を行う
   - 入力項目（状態、フェーズ、工数、レベル、説明）のバリデーションを実装する

6. ロック表示と操作制御
   - ロック状態（未ロック／自分ロック／他人ロック／期限切れ）の表示を実装する
   - ロック状態に応じて行クリックや編集ボタンの活性／非活性を制御する

7. Excel 出力・バックアップ UI
   - Excel 出力ボタン押下時のフィードバックとダウンロードトリガーを実装する
   - DB バックアップモーダルの表示・閉じる処理を実装する

8. UX 仕上げ
   - ローディング、エラー表示、トーストなどを追加して操作感を整える
   - レイアウト・レスポンシブ挙動を最終調整する

## バックエンドの実装順

1. Prisma スキーマとマイグレーションの確定
   - `schema.prisma` と実データベース（`migration.sql`）の整合を確認する
   - 開発用 DB にマイグレーションを適用し、基本テーブルを作成する

2. 認証基盤（NextAuth）整備
   - 資格情報プロバイダとユーザーテーブルの接続を実装する
   - セッションにユーザー ID とロール（admin / viewer）を含める

3. タスク CRUD サーバーアクション
   - `getTasks`, `createTask`, `updateTask`, `deleteTask` を Prisma ベースで実装する
   - バリデーションや権限チェックを共通処理として切り出す

4. ロック関連処理
   - `lockTask`, `unlockTask` などロック操作用のアクションを実装する
   - ロック期限切れの判定ロジックを実装し、自動解除の方針を決める

5. 集計・統計用クエリ
   - 完了数、未着手数、進行中数、総工数などの集計クエリを実装する
   - 一覧画面用にまとめて取得するヘルパーを用意する

6. Excel 出力 API
   - タスク一覧から Excel/CSV を生成するエンドポイントまたはアクションを実装する
   - フロントからのパラメータ（フィルタ条件など）が渡せるように設計する

7. DB バックアップ機能
   - `pg_dump` 等を呼び出すバックアップ処理のラッパーを実装する
   - 実行権限を admin のみに制限し、結果をダウンロードまたは保存できるようにする

8. ログ・エラーハンドリング強化
   - 主要なサーバーアクションにログ出力とエラーハンドリングを追加する
   - 運用時に問題箇所が追いやすいようにする
